# Общее описание

На решение задачи отводится строго одна пара в соответствии с расписанием.

Для решения задачи необходимо написать программу, отладить и протестировать ее. Проверка исходных данных на допустимость **обязательна**. Допустимо использовать заголовочный файл, разработанный для лабораторной работы 1.

Все данные передаются в функции как параметры, использование глобальных переменных недопустимо.

Для каждого задания предоставлены заготовки решений с заголовочным файлом с заголовком функции, а также заготовка main.cpp.

Просьба не использовать компилятор по умолчанию из Visual Studio. Можно, например, поставить minigw, clang, gcc и использовать Clion от Jetbrains.

Компиляцию программы необходимо производить с использованием CMake.

# Загрузка КР

Результаты контрольной работы загружаются в отдельную папку в репозиторий, который используется при сдаче лабораторных работ. Необходимо создать отдельную папку с названием "CW. 3.01". В этой папке для каждого задания необходимо загрузить Ваше решение, включая: набор файлов для сборки (*CMakeLists.txt*), исходный код (*.cpp, .hpp*), корректно заполненный файл *.gitignore*. Система сборки должна осуществлять сбор всех исполняемых файлов единовременно (как было показано на семинаре 3.06).

Тестовые примеры необходимо добавить в виде отдельных файлов – файл с исходными данными и результатом (для всех возможных случаев, рассматриваемых в программе). В начале файла в виде комментария запишите полное условие задачи.

Загрузить результаты выполнения КР необходимо не позже чем через 10 минут после окончания пары в соответствии с расписанием.

## Описание критериев

1. **Алгоритм (0,5).** Критерий оценивается по следующим составляющим; при невыполнении одного из пунктов оценка по критерию снижается:

|Составляющая|Что проверяется|
|-|-|
|**Вспомогательные структуры**|Отсутствие вспомогательных массивов и матриц, кроме заданных в условии (A, B, C и т.д.). Допускаются только скалярные переменные (счетчики, индексы, промежуточные значения).|
|**Тип массивов**|Использование стандартных массивов и матриц фиксированного размера (например, `int A[4];`, `int B[2][2];` или с константами `MAX_ROWS`, `MAX_COLS`). Не использовать std::vector и другие контейнеры STL в качестве замены массивов из задания.|
|**Заголовочный файл из ЛР 3.01**|Использование функций из заголовочного файла, разработанного в ЛР 3.01 (поиск в массиве, минимум/максимум и т.п.), где это применимо к заданию.|
|**Способ обращения к элементам**|В задании 1 — обращение к элементам матрицы и массивов **по индексу** (A[i][j], B[k], C[m]). В задании 2 — обращение к элементам массива **с использованием указателя** (например, *(A + i)).|
|**Соответствие условию**|Логика программы полностью соответствует формулировке задания: корректный результат для всех описанных случаев, выполнение требований «оформить в виде функций с параметрами», «дополнительный массив не использовать» и т.п.|
|**Параметры и глобальные переменные**|Все данные передаются в функции только через параметры; использование глобальных переменных недопустимо.|

1. **Сборка программы (0,1).** Корректно описанный файл *CMakeLists.txt* (или набор таких файлов). При сборке с использованием CMake проект успешно конфигурируется и собирается без ошибок; все исполняемые файлы заданий собираются единовременно (одной целью или через add_subdirectory).

2. **Работа программы (0,1).** Оценивается следующее:

    - программа дает корректный результат на допустимых входных данных;

    - выполняется проверка исходных данных на допустимость (размеры, указатели, границы), некорректный ввод обрабатывается без падения;

    - отсутствуют утечки памяти (динамическая память при необходимости освобождается, либо не используется).

1. **Комментирование DoxyGen (0,1).** Все файлы с исходным кодом (.cpp, .hpp) содержат комментирование в формате DoxyGen (на библиотеку из ЛР 3.01 это не распространяется).

    - **К комментарию к файлу** — секции: **file**, **brief**, **details**, **author**, **date**. В main.cpp в блоке к файлу дополнительно указывается полное условие задания для выполнения.

    - **К комментарию к функции** — секции: **brief**, **details**, **param** (для каждого аргумента), **return**, **author**, **date**.

1. **Тесты (0,1).** Предоставляется текстовый файл (или файлы) с набором входных и выходных данных. Должны быть рассмотрены **все существенные случаи** работы программы, в том числе: пустые или нулевые размеры, один элемент, граничные значения, случаи «ни одного подходящего элемента» и «все элементы подходящие», некорректные индексы (если применимо). В начале файла в виде комментария указывается полное условие задачи.

2. **Загрузка в репозиторий (0,1).** Правильность загрузки: папка с названием «CW. 3.01»; для каждого задания — исходный код (.cpp, .hpp), CMakeLists.txt, .gitignore; в репозитории **отсутствуют** исполняемые файлы, папки сборки (build, cmake-build-* и т.п.), объектные файлы (.o, .obj), папки IDE и внешних приложений.

# Пример задания

# Вариант 10

1. Дана матрица A[0:rowsA-1][0:colsA-1] и массив B[0:lenB-1]. Разработать программу, которая вычисляет массив C, состоящий из элементов матрицы A, расположенных перед минимальным элементом каждого столбца, если сумма этих элементов (расположенных строго перед минимальным) содержится в массиве B. Вычисления оформить в виде отдельных функций с параметрами. Обращение к элементам матрицы и массива — по индексу.

2. Дан одномерный массив целых чисел A[0:lenA-1]. Удалить путем сдвига все элементы, являющиеся полными квадратами. Дополнительный массив не использовать. Обработку массива оформить в виде функции с параметрами. Обращение к элементам массива — с использованием указателя.
